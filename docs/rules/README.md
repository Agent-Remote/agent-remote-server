# Rules Index

This directory is the local source of truth for repository standards.

| File | Scope |
| --- | --- |
| `01-project-overview.md` | Project purpose, current boundary, non-goals |
| `02-architecture.md` | Runtime layout, module boundaries, dependency direction |
| `03-tech-stack.md` | Python version, framework, runtime dependencies |
| `04-code-style.md` | Formatting, typing, imports, naming |
| `05-comment-style.md` | Chinese docstrings, inline comments, Pydantic field descriptions |
| `06-commands.md` | Approved local commands |
| `07-quality-security.md` | Quality gates and security constraints |
| `08-collaboration.md` | Git hooks, commits, branches, PR expectations |
| `09-data-persistence.md` | Database, Redis, Alembic, secrets |

Agents and contributors must read the relevant rule files before changing code.
